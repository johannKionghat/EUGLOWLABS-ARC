### Lessons learned

- Pour tout **script Bash défensif avec `set -e`** : ne JAMAIS utiliser `((var++))` sur variable initialisée à 0 (post-increment retourne 0 → trigger `set -e`). Pattern correct : `var=$((var + 1))`.
- Pour les **ranges `Internal: true`** (Docker network) : tester config (`docker inspect`) ET runtime (ping depuis container). Une seule vérif rate des bugs subtils.
- Pour la **doc utilisateur** : la structure `(mandatory)` vs `(optionnel)` en TOC doit être visible dès la lecture, pas enfouie en sous-section. UX drastique.
- Pour les **commandes copy-paste** : utiliser des exemples concrets (`app.example.com`) au lieu de placeholders (`<your-domain>`). Évite les bugs UX.
- Pour les **regex de validation** (commit-msg, etc.) : anticiper les scopes alphanumériques dès le départ (`[A-Z][A-Z0-9]*`) plutôt que `[A-Z]+`. Coût initial nul, évite les fix orthogonaux plus tard.
- Pour le **troubleshooting d'une procédure** : ne pas inventer des cas théoriques, documenter UNIQUEMENT ce qu'on a vu en réel pendant le développement. Plus authentique, plus utile.
