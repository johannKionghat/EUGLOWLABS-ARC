## Bilan

### Ce qui a bien marché

- Découpage en **3 sous-tâches avec cadrage figé en amont** (S1-S6, B1-B8, V1-V3) → 0 friction de design.
- Pattern **« diff proposé → user valide → application »** maintenu sur 001c, **déviations signalées proactivement** par Claude Code (rclone obscure, auto-discovery container).
- **Sandbox livré simplement** (1 task `loop` sur `arc_networks`) — design clean.
- **Auto-détection du bug `.gitignore`** par Claude Code AVANT push — le rôle `backups` aurait été perdu sans le fix exception `!packages/arc-cli/playbooks/roles/backups/`.
- Pattern **stat `r2.env` + skip défensif** pour backups → permet à un user sans R2 de tester `arc setup --apply` sans bloquer.
- **Block conditional Ansible** pour cleanliness (vs `when:` dupliqué sur 9 sub-tasks).
- `no_log: true` sur 4 tasks sensibles (parse, 2× obscure, template `rclone.conf`) → pas de leak credentials dans les logs Ansible.
- **Auto-discovery container Postgres** via Docker compose labels (`--filter project=localai service=db`) → robuste aux changements de naming Compose v2.
- **12 CLI gaps capturés** (6 nouveaux 001c + 6 hérités) → traçabilité de la dette.

### Ce qui a posé problème

- **Découverte `rclone obscure` mandatory** pendant le diff (pas dans le brief initial). Sans cette correction, le crypt remote aurait planté au premier run en E2E-001.
- **Container Postgres pas `supabase-db`** comme dans le brief initial — Compose v2 avec `-p localai` produit `localai-db-1`. Auto-discovery par labels = fix idiomatique mais aurait pu être anticipé en cadrage 001b si les containers de `start_services.py` avaient été listés.
- **`.gitignore:38 backups/` générique** a failli ignorer le rôle entier — détecté à temps par Claude Code, fix par exception explicite. Signal CLI gap : auditer `.gitignore` pour patterns trop génériques.
- **Parse `r2.env` avec `slurp` + `loop-combine`** est plus complexe que prévu (pattern idiomatique Ansible 2.10+ mais nécessite filtre regex défensif).
- **~50 min réaliste** pour la sous-tâche 2 backups vs 40 min annoncés (7 fichiers + 3 templates + parse logic + obscure dance + auto-discovery).

### Lessons learned

- Pour **tout outil S3-compatible avec encryption** (rclone crypt, restic, etc.), vérifier si les passwords doivent être obscurcis/dérivés AVANT de cadrer le rôle. Sinon découverte runtime garantie.
- **Auto-discovery via Docker labels** (`--filter label=...`) > hardcode container name. Pattern à généraliser pour toute interaction avec containers d'un compose tiers.
- `.gitignore` patterns génériques (`backups/`, `logs/`, `tmp/`) doivent avoir des **exceptions explicites** pour le code source. Audit recommandé.
- **Block conditional Ansible** avec `when:` au niveau du block est plus lisible que `when:` dupliqué sur 9 sub-tasks. Pattern à utiliser dès qu'on a 3+ tasks gated par la même condition.
- `no_log: true` mérite une discrimination fine — sur les tasks qui exposent les valeurs, pas sur celles qui exposent juste les paths. Trop de `no_log` = debug runtime impossible.
- Pour les rôles avec **credentials operator-managed** (R2, JWT, etc.), le pattern stat-pivot + skip défensif + warning loud est meilleur que fail rôle (UX user-friendly).

### Décisions structurantes prises

- **ADR-0008 implémenté** : 3 réseaux Docker (`prod_net 172.20/24` external, `ai_net 172.21/24` external, `sandbox_net 172.22/24` **INTERNAL** anti-exfiltration).
- **Subnets fixes** (vs auto-assigné) pour traçabilité et debug.
- **Labels `arc.network` / `arc.role` / `arc.managed-by`** sur tous les networks ARC.
- **rclone via apt** (pas curl|sh) — cohérent ufw/fail2ban pattern.
- **Encryption MANDATORY** via rclone crypt remote (zéro plaintext sur R2).
- Backup script Bash dans `/usr/local/bin/arc-backup.sh`, mode 0750.
- **Cron classique** `/etc/cron.d/arc-backup` (pas systemd timer pour MVP).
- **6 sections de backup** : Coolify data + ARC credentials + state.json + Postgres ai-stack + cleanup local logs + cleanup remote retention.
- **Retention 30 jours** (local + remote, configurable via `arc_backup_retention_days`).
- **Auto-discovery container Postgres** via Docker compose labels (pattern robuste cross-renaming).
- **Pattern parse `r2.env`** : `ansible.builtin.slurp` + `set_fact` loop-combine avec regex filter défensif.
