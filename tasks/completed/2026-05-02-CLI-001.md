## Bilan
- **Ce qui s'est bien passé** : plan en 6 sous-tâches respecté, tests Vitest collocated qui marchent du premier jet (3/4) puis 4/4 après ajustement. Smoke test du binaire compilé OK direct. CI verte premier essai (Node 30s + Go 25s).
- **Ce qui a posé problème** :
  - tsconfig `types: []` initial bloquait la résolution de `process` → ajout `@types/node` + `types: ["node"]`. Pas pris en compte dans le plan initial — leçon : pour tout package qui touche au runtime Node, prévoir `@types/node` dès la création du tsconfig.
  - Test "unknown command" attendait stderr non vide ; clipanion 4 RC4 route ses erreurs sur stdout. Assertion relâchée à `stdout+stderr ≠ ""`. Comportement à reverifier si on monte en clipanion stable plus tard.
- **Ce qu'on a appris** : clipanion 4 RC4 est stable et propre côté API ; `binaryLabel`/`binaryName`/`binaryVersion` produisent une UX d'aide colorée out-of-the-box. La factory `buildCli()` + helper `runFromArgs()` est un excellent pattern à reprendre pour toutes les commandes futures.
- **Décisions prises** : aucune ADR-level. Conformes à ADR-0002 (clipanion acté, single-binary reporté à CLI-025).
  - VERSION en dur dans `src/version.ts` (synchro manuelle avec `package.json#version` jusqu'à CLI-025)
  - Shebang `#!/usr/bin/env node` (pas Bun) — Node est la baseline `.tool-versions`
  - `runExit` jamais utilisé dans les tests (casse Vitest), `cli.run` direct via `runFromArgs`
