## Bilan
- **Ce qui s'est bien passé** : 6 sous-tâches respectées. Découpage `errors.ts` / `load.ts` / fixtures / tests propre. CI verte premier essai (Node 32s + Go 26s).
- **Ce qui a posé problème** :
  - `zod` manquait en dep directe d'arc-cli (était transitive via arc-shared) — typecheck a planté sur l'import du type `ZodError`. Fix : `pnpm add zod`. Pas dans le plan initial.
  - `Array.reduce<string>` n'inférait pas correctement les paramètres avec le typage zod 4 de `issue.path`. Remplacé par boucle `for...of` — plus simple, identique runtime.
- **Ce qu'on a appris** : pour tout package qui consomme un type d'une lib transitive, ajouter la lib comme dep directe dès le départ — la résolution de types ne suit pas la chaîne npm comme on pourrait le croire. À documenter dans `coding-style.md` plus tard si on rencontre le problème ailleurs.
- **Décisions prises** : aucune ADR-level. Toutes locales et réversibles.
  - Lib **`yaml`** (eemeli/yaml) — erreurs riches avec positions
  - **Throw `ConfigError`** plutôt que Result type (idiome Node, simple)
  - **Async only** (pas de version sync)
  - **Pas d'interpolation `${VAR}`** — reportée à une tâche dédiée si elle apparaît dans le backlog
