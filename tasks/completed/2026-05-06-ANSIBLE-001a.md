## Bilan

### Ce qui a bien marché

- Cadrage figé en **8 décisions UX/sécurité avant code** (Q1–Q8) → 0 friction de design sur les sous-tâches.
- Découpage en **5 sous-tâches < 30 min chacune** → respect règle CLAUDE.md « 1 PR < 2h ».
- Pattern **« Claude Code propose le diff → user valide → application »** → 0 erreur livrée en production.
- **Safety check `authorized_keys`** avant SSH key-only → anti-lockout solide.
- **`sshd -t` avant restart** → garantie zero-downtime SSH (validate-before-flush via ordre task/handler intentionnel).
- **Mapping arch `x86_64` → `amd64`** anticipé → support arm64 gratuit, pas de hardcoding amd64.
- **`append: true` obligatoire** sur groupe docker → préservation sudo et autres groupes (sinon perte sudo).
- **`ansible-lint` en validation finale** → 17 → 0 violations, profile escaladé `min` → `production`.
- **Allow-before-enable strict** sur UFW → impossibilité de lockout réseau pendant l'exécution.

### Ce qui a posé problème

- **Setup environnement initial : ~1h30 imprévue**
  - Repo dans OneDrive → conflits potentiels avec WSL → déplacement vers `C:\Dev\`.
  - `pnpm install` échec EACCES sur `/mnt/c/` via WSL2 → clone dans WSL home ext4 natif.
  - Authentification GitHub PAT à reconfigurer dans WSL.
- **Claude Code initialement dans Git Bash Windows :**
  - Auteur de commit `DESKTOP-ACDD6TV\\User` au lieu de `johannKionghat`.
  - Hook `lefthook` bypassé silencieusement.
  - Détecté via auteur incorrect sur `b62ed47` → réinstallation Claude Code dans WSL natif.
- **Régression git** via `reset --soft` + `amend` mal positionnés → réparation via `reflog`.
- **Script `validate-commit-msg` ne supportait pas le suffixe `a/b/c`** → fix `INFRA-006` (commit `a932882`).
- **`ansible-lint` absent du WSL** → install manuelle requise (sudo non-interactif côté Claude Code).
- **`arc_user` expression initialement divergente** entre rôles `hardening` et `docker` → aligné en sous-tâche 4 sur l'expression `lookup('env', 'SUDO_USER') | default(lookup('env', 'USER'), true)`.
- **Régression TS4114 (`noImplicitOverride`) sur `ComposeDirAccessError.cause`** détectée en sous-tâche 5 d'ANSIBLE-001a. Origine : INSTALL-002 commit `4e59abd`. Fix orthogonal commité séparément avec tag `[INSTALL-002]` (commit `a63ecd1`). Le critère « typecheck globaux verts » avait initialement été coché par erreur (seul `pnpm lint` avait été lancé en sous-tâche 5).

### Lessons learned

- **Toujours dev dans WSL ext4 (`~/`), jamais dans `/mnt/c/`** (perf I/O 10× + idempotence pnpm).
- **Sortir le repo de OneDrive** est non-négociable pour dev WSL stable.
- **Vérifier l'environnement Claude Code avant chaque sous-tâche** (`pwd`, `which node`, `git config user.name`).
- Le **format de TASK-ID doit anticiper le suffixe `a/b/c`** dès la 1ère décision de découpage.
- Les **tâches Ansible avec impact réseau** (UFW, sshd) demandent des safeguards explicites :
  - UFW : allow before enable
  - sshd : `-t` validate before reload
  - `authorized_keys` : check before disabling password auth
- **Mapping `ansible_architecture` (uname -m) → Debian arch (`amd64`/`arm64`)** : pas évident, à pinner.
- **`pnpm typecheck` doit être lancé séparément** de `pnpm lint` et `pnpm test` lors d'une validation finale — ne jamais se contenter de la run partielle même si Turbo cache hit.

### Décisions structurantes prises

- **Aucun nouvel ADR** (les ADR-0008 / 0011 / 0012 / 0015 couvraient déjà le périmètre Ansible).
- **Convention de suffixe `a/b/c`** officialisée pour découpage de tâches > 2h (cf. INFRA-006 commit `a932882`).
- **`arc_user` résolu via `SUDO_USER || USER`** (pas `ansible_user_id` qui retourne `root` sous `become: true`).
- **Ports UFW** : 22 (`{{ arc_ssh_port }}`) + 80 + 443 only. **PAS 8000** (Coolify dashboard reste localhost via SSH tunnel).
- **fail2ban** : `sshd` + `recidive` uniquement (pas apache/nginx en 001a).
- **unattended-upgrades** : security only + no auto-reboot (Q8).
- **Docker via keyring moderne** `/etc/apt/keyrings/docker.asc` (pas `apt_key` deprecated).
- **`yes`/`no` → `true`/`false`** (alignement YAML 1.2 spec via ansible-lint cleanup).

## Objectif
Poser la fondation Ansible d'ARC (structure `roles/`, conventions, invocation depuis `setup.yml`) et livrer les **deux premiers rôles système** :
- `hardening` — UFW limité (22 + 80 + 443), fail2ban (jail `sshd`), SSH key-only, `unattended-upgrades`.
- `docker` — Docker Engine + plugin Compose v2, ajout de l'user qui pilote ARC au groupe `docker`.

Ces rôles préparent la machine cible pour `coolify` / `ai-stack` (ANSIBLE-001b) et `sandbox` / `backups` (ANSIBLE-001c). Le stub `playbooks/setup.yml` reste fonctionnel : à la fin de 001a, il invoque les rôles `hardening` + `docker` (et plus seulement le `debug` no-op).

## Scratchpad — Décisions actées avant code (2026-05-05, validées utilisateur)

- **Q1 — Port SSH** : 22 par défaut, variable `arc_ssh_port` exposée. Règle UFW utilise `{{ arc_ssh_port }}` pas `22` en dur. Gestion transition open-new-before-close-old → 001b/c.
- **Q2 — `become`** : `become: true` global au play en 001a. 001b/c raffineront par rôle si nécessaire.
- **Q3 — Distro** : `assert ansible_os_family == "Debian"` (Ubuntu + Debian). Warning non-bloquant si distro != Ubuntu 24.04.
- **Q4 — Docker Compose** : plugin v2 uniquement (`docker-compose-plugin`). Test post-install : `docker compose version` retourne `2.x.x`. Pas de purge auto de v1 résiduel.
- **Q5 — UFW IPv6** : `IPV6=yes` dans `/etc/default/ufw`.
- **Q6 — Ports UFW** : 22/80/443 uniquement. Pas d'ouverture 8000 (Coolify dashboard reste localhost-only, accès SSH tunnel).
- **Q7 — fail2ban jails** : `sshd` + `recidive` uniquement en 001a. `maxretry=5`, `bantime=1h`, recidive `bantime=1 semaine`.
- **Q8 — unattended-upgrades** : security only. `Automatic-Reboot "false"`. `Allowed-Origins` limité à `${distro_id}:${distro_codename}-security`.
