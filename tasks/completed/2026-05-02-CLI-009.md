## Bilan
- **Ce qui s'est bien passé** : 4 sous-tâches respectées, design d'interface propre, `MockAdapter` minimaliste avec helpers `programExec` + `seedFile`. Tests verts du premier coup. CI verte premier essai (Node 37s + Go 24s). Le pattern callback `onChunk` est plus simple à tester qu'un `EventEmitter`.
- **Ce qui a posé problème** : rien.
- **Ce qu'on a appris** : avoir un mock dans le package dès la pose de l'interface est productif — CLI-012/015/016 pourront tester leurs commandes contre `MockAdapter` sans toucher au shell réel. Pattern à reprendre quand on définira d'autres interfaces transverses (ex: `NotificationChannel` plus tard).
- **Décisions prises** : aucune ADR-level. Toutes alignées avec ADR-0009.
  - Localisation `src/exec/` (plus précis que `src/adapters/`)
  - `copyFile` opérateur → adapter (pas de `downloadFile`, `readFile` couvre)
  - Callback `onChunk` plutôt qu'`EventEmitter` ou async-iterator
  - `describe()` libre forme plutôt qu'enum
  - Interface dans `arc-cli` (pas `arc-shared`)
  - `MockAdapter` exporté
