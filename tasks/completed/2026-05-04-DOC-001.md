### CLI gaps identifiés (matière pour CLI-029+ post-DOC-001)
La rédaction de §1.a et §1.b a fait apparaître des opérations qui devraient à terme être scriptables via `arc` mais ne le sont pas :

- `arc project set-repo <PROJECT_NAME> --git=<URL> [--branch=main]` — connecter un repo Git au projet Coolify (actuellement passe par UI Coolify étape 5).
- `arc project set-env <PROJECT_NAME> --file=<env-file>` — injecter en lot un fichier d'env vars (actuellement passe par UI Coolify étape 5, copier-coller manuel).
- `arc project add-domain <PROJECT_NAME> <CUSTOM_DOMAIN>` — ajouter un domaine custom (actuellement UI Coolify étape 7).
- `arc db restore <PROJECT_NAME> --from=<dump-path> [--clean]` — wrapper autour de `docker exec ... pg_restore` qui détecte automatiquement le container Postgres et choisit la bonne base (actuellement procédure manuelle étape 4 §1.a et §1.b).
- `arc storage migrate <BUCKET_NAME> --from=<remote-supabase-url> --token=<key>` — wrapper autour de `supabase storage cp` + `rclone` (actuellement procédure manuelle étape 5 §1.b).
- `arc supabase keys` — afficher `<ARC_JWT_SECRET>`, `<ARC_ANON_KEY>`, `<ARC_SERVICE_ROLE_KEY>` (actuellement `cat ~/.arc/arc.config.yml | grep` + `docker exec` à tâtons étape 1 §1.b).
- `arc project move <PROJECT_NAME> --to=<VPS_B_IP>` — wrapper §2 (orchestration backup partiel + scp + restore).
- `arc clone --from=<VPS_PROD_IP> --domain=<DOMAIN_STAGING>` — wrapper §3 (clone full + remap domaines/secrets).
- `arc supabase rotate-jwt` — wrapper §3 4.b Option 1 (génère secret + redéploie stack + récupère nouvelles clés + invalide tous les tokens existants).
- `arc setup --reconfigure` — relire `~/.arc/arc.config.yml` modifié et appliquer les changements idempotemment (referencé par §3 4.b Option 1, mais déjà partiellement couvert par INSTALL-001 idempotence).

À considérer pour un nouveau lot de tâches `CLI-029` à `CLI-038` à proposer une fois DOC-001 complétée et la phase 1.5 en cours de validation.
