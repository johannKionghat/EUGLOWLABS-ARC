## Bilan
- **Ce qui s'est bien passé** : plan en 6 sous-tâches respecté, 6/6 cas de test du hook conformes du premier coup, lefthook installé via `prepare` script sans accroc, CI verte au premier essai.
- **Ce qui a posé problème** : le passage de `pnpm lint:fix` a fait du churn massif CRLF↔LF sur ~14 fichiers (Windows + git autocrlf + Biome lineEnding LF). Découvert pendant la sous-tâche 6.
- **Ce qu'on a appris** : sur un monorepo dev sous Windows, `.gitattributes` avec `* text=auto eol=lf` est obligatoire dès le bootstrap. À considérer pour les futurs monorepos. Aussi : `git add --renormalize .` après ajout de `.gitattributes` enlève le churn d'un coup.
- **Décisions prises** : aucune ADR-level. Outillage local pur.
  - lefthook plutôt que husky (Go natif, déclaratif, plus rapide)
  - validateur Node maison plutôt que commitlint (zero-dep, messages d'erreur sur-mesure)
  - bypass list explicite (Merge, Revert, fixup!, squash!, amend!) + suffixe GitHub squash `(#NN)`

## Dépassement de scope (justifié)
Ajout de `.gitattributes` (LF enforcement) hors plan initial. Justification : friction Windows découverte pendant la tâche, fix structurel d'1 ligne qui évite des heures de churn ultérieur. Documenté dans le commit et dans la PR.
